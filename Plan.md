# 1. Перечень автоматизируемых сценариев
## 1.1. Оплата тура дебетовой картой
Входные данные:
А. Валидные. Номер карты: любое 16-тизначное число (4444 4444 4444 4441 - операция одобрена банком,
4444 4444 4444 4442 - операция отклонена банком); Месяц: 01, 02, 03, 04, 05, 06, 07, 08, 09, 10, 11, 12; 
Год: 22, 23, 24, 25, 26, 27, 28;  Имя: любые буквы латиницей и кириллицей, пробел и знак "-"; Код CVC/CVV: любое трехзначное число.
**_Позитивные сценарии:_**
Предварительные действия: Пользователь нажимает кнопку "Купить".

1. Пользователь вводит валидные данные (номер карты 4444 4444 4444 4441). Нажимает кнопку "Продолжить".
**Ожидаемый результат: пользователь видит на экране сообщение "Успешно! Операция одобрена банком.".**
2. Пользователь вводит валидные данные (номер карты 4444 4444 4444 4442). Нажимает кнопку "Продолжить".
**Ожидаемый результат: пользователь видит на экране сообщение "Ошибка! Банк отказал в проведении операции.".**

**_Негативные сценарии:_**
1. Пользователь вводит невалидные значения в поле "Номер карты" (цифру, 15-значное число, 17-значное число, символы, буквы латиницей, буквы кириллицей). Пользователь вводит валидные значения в остальные поля формы. Нажимает кнопку "Продолжить".
**Ожидаемый результат: нижняя граница поля "Номер карты" становится красной, под ней пользователь видит надпись "Неверный формат".**
2. Пользователь оставляет пустым поле "Номер карты". Пользователь вводит валидные значения в остальные поля формы. Нажимает кнопку "Продолжить".
**Ожидаемый результат: нижняя граница поля "Номер карты" становится красной, под ней пользователь видит надпись "Поле обязательно для заполнения".**
3. Пользователь вводит невалидные значения в поле "Месяц" (цифру, 3х-значное число, символы, буквы латиницей, буквы кириллицей). Пользователь вводит валидные значения в остальные поля формы. Нажимает кнопку "Продолжить".
**Ожидаемый результат: нижняя граница поля "Месяц" становится красной, под ней пользователь видит надпись "Неверный формат".**
4. Пользователь вводит невалидные значения в поле "Месяц" (2х-значное число больше 12). Пользователь вводит валидные значения в остальные поля формы. Нажимает кнопку "Продолжить".
**Ожидаемый результат: нижняя граница поля "Месяц" становится красной, под ней пользователь видит надпись "Неверно указан срок действия карты".**

# 2. Перечень используемых инструментов
- Язык написания тестов - Java
- Среда разработки - IntelliJ IDEA Ultimate
- Клиент для работы с БД - DEA SQL Client
- Docker Compose для запуска контейнера с базой данных
- Selenide для тестирования веб-интерфейсов (переход на страницу курса, заполнение формы)
- Faker для генерирования случайных пользователей
- Appveyour для использования CI
- Allure для формирования отчета об ошибках

# 3. Перечень и описание возможных рисков при автоматизации
- Изменение дизайна и верстки страниц могут привести к изменению селекторов, используемых в тестах
- Удаление / добавление полей в форму: тесты падают / уменьшается охват тестирования отправки формы
- Трудности с поиском локаторов тестируемых элементов на страницах 
- Недстаточная квалификация и мотивированность тестировщика-автоматизатора может привести к затягиванию процесса автоматизации и написанию тестов с ошибками

# 4. Интервальная оценка с учётом рисков (в часах)
- планировании автоматизации тестирования - 4 часа
- написание автотестов - 30 часов
- подготовке отчётных документов по итогам автоматизированного тестирования - 1 час
- подготовка отчётных документов по итогам автоматизации - 4 часа
